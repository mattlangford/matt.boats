Blog Creation

First post, please ignore

I've been working on the format for this blog for a little bit. I figured it'd make a good first post!

I'm aiming for a couple things with this:

  1. Very simple interface
  2. Statically hosted with no javascript
  3. Low overhead for writing shortish blogs
  4. It's something to do

Layout

The general idea is that I'll maintain a set of markdown files -- one for each post. A python script (AI generated make_page.py) can scrape those files and generate a set of html pages. The index page is also gneerated, by collecting the title from the markdown pages and some metadata at the top of the file.

I've elected to keep each blog post in the root folder, or if there is associated data, in folders that follow an arbitrary 001_name structure.

I suspect it will be hard to keep the layout working on different devices and browsers (including light/dark mode). I'll only be testing on Safari on my Mac.

Testing

Here are some examples that I use to test the markdown generation. There are code snippets

def get_hello_world():
    return "hello world!"

print(get_hello_world())

and inline code snippets. I've got lists of all kinds, like numbered

  1. Number 1
  2. Number 2

or bulletted

  • Bullet 1
  • Bullet 2

I can link to internal things, or link to external ones.

Images should work too...

Hosting

The deployable pages are packaged into the dist/ folder, and pushed to the deploy branch on the repo. Github pages knows to watch this branch and will host the data from there. I managed to set up a custom domain through namecheap and hookup the cname records so it points to the hosted pages.
